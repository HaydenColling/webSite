File Integrity Monitor

At the start of the script, I made a check for if the user has run the script before. I created a variable to check if the file where we are storing the data exists.


If it exists, it will overwrite the existing data and set the header of the file. If it doesn't exist, it will create a folder with a text file with the header information. The functionality I provided with this script allows for the user to add multiple files to monitor. If the file exists, it will alert the user and bring them back to the options.



Once the user has added all the files it will begin to monitor the MD5 hash.



To get the hash of the file I used the read function and encode function to create a single variable. Therefore, allowing hashlib.md5 function to create the hash. Once the hash is created, we open the file where all the hashes are located and add it to the text file.

The data structure used to store the file locations and the hashes is the dictionary due to the need for a key-value pair functionality. Once we have finished acquiring all the hashes, we continuously gather the hashes of the targeted files. If the program detects a change in the hash, it will alert the user in the command line.
